# Contributing to RestoLogic Enterprise OS

We love community contributions! Whether it's fixing a bug, improving documentation, or proposing a new feature, your input helps us make RestoLogic better for everyone in the hospitality industry.

## How to Contribute

1. **Fork the Repository**
   - Click "Fork" at the top right of the repo to create your own copy.

2. **Clone Your Fork**
   - Download your fork to your local machine.
   - `git clone https://github.com/YOUR-USERNAME/restologic-enterprise-os.git`

3. **Create a New Branch**
   - `git checkout -b feature/YourFeatureName`

4. **Make Changes**
   - Be clear and descriptive in code and comments.
   - Follow any style guidelines found in the repo.

5. **Commit Changes**
   - `git commit -m "Add feature: description"`

6. **Push to Your Fork**
   - `git push origin feature/YourFeatureName`

7. **Open a Pull Request**
   - Describe what you changed and why.
   - Link any related issues or feature discussions.

## Code Standards

- Write clear, readable code.
- Use English for all comments and documentation.
- Keep commits focused—one feature or fix per pull request.
- If you see a need for a major refactor or architectural change, open an issue before beginning.

## Reporting Issues

- Please use the [GitHub Issues](https://github.com/restologicllc/restologic-enterprise-os/issues) feature.
- Include steps to reproduce, screenshots, and environment info if possible.

## Community

By participating, you agree to our [Code of Conduct](CODE_OF_CONDUCT.md).
